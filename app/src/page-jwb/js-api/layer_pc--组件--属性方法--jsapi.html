基础属性

注意：这里只列举一些基础的公用的属性，详细的属性请移步layer官网。后面方法特有的属性会在内置方法里面做详细解释
title - 弹窗的标题
类型：String 默认：弹窗标题 ， 这个默认值是自己设定的，用的时候依据实际需求填写吧！
说明： 弹窗的标题
content - 弹窗内容
类型：String/DOM 默认：主体内容 ， 这个默认值是自己设定的，用的时候依据实际需求填写吧！
说明： 弹窗内容，强调一下DOM，这里是实现弹窗自定义的关键。通过定义一个html模板，然后将它的内容传递过来。
skin - 皮肤/样式类名
类型：String 默认：layer-ext-jwb/layer-ext-jwb_ask ，
说明： 这个是根据layer自定义样式接口文档自行定义的，一般不用变动。（除了不用预先设定好的样式情况外，要是真出现这种情况还是别用该组件了，赶紧去用layer.open()方法自行coding去吧，天啊撸！）
area - 宽度和高度
类型：Array 默认：["525px", "400px"]/["386px", "252px"]
说明： 弹窗的宽度和高度（包括标题部分），左边是宽度值右边是高度值，这个属性的默认值也是根据项目自行设定的但是不是一成不变的，如果调用内置方法layerPc.page()/layerPc.frame()则需要根据UI给的设计稿自行传值，后续会做解释。高度超出的部分会出滚动条！
btn - 底部操作按钮
类型：Array 默认：["确定"]/["确定", "取消"]
说明： 底部操作按钮，根据设计图给的进行传值

内置方法

注意：由于产品用户体验的需求，组件只需在骨架内引入即可,所以要是想在iframe页面里面调用组件的方法前面需要加parent

在骨架里调用组件方法

//在骨架内调用initTab方法--我现在就是个练习的靶子，后面会详细介绍我的，嘻嘻嘻！
layerPc.alert();

在iframe里调用组件方法

//在骨架内调用initTab方法--我现在就是个练习的靶子，后面会详细介绍我的，嘻嘻嘻！
parent.layerPc.alert();
layerPc.page(params) - 页面层
参数一：
类　型：Object
解　释：该参数有N个属性
1.不变的属性（不需要传值）: skin 使用自带的默认值即可
2.title: 弹窗的标题 默认：页面层的标题 类型：String
3.content: 弹窗的内容 默认：页面层的内容 类型：String/DOM
4.area: 弹窗的宽度和高度，第一个值是宽度，第二个值是高度。 默认：["525px", "400px"] 类型：Array
5.N+: layer最底层的基础属性（除了上面出现的） 
说　明： 这个是最为简单但是是最为底层的方法了，为什么这么说呢，因为这个方法可以接受N多个参数，不像其它方法那样有具体的固定值。可以自行根据UI和需求自行发挥，可以说是变化多端的可高度自定义的方法了！ 
		layerPc.page({
		   title: '标题',
		   content: '我是光光的但是非常底层的可高度自定义的page层,'
		});
layerPc.alert(params) - 一个按钮的提示框--通知框
参数一：
类　型：Object
解　释：该参数有6个属性
1.不变的属性（不需要传值）: skin/area 使用自带的默认值即可
2.title: 弹窗的标题 默认：弹窗标题 类型：String
3.content: 弹窗的内容 默认：主体内容 类型：String/DOM
4.btn: 弹窗的底部操作按钮，数组传递一个值即可 默认：["确定"] 类型：Array
5.call: 弹窗的底部操作按钮的回调函数 默认：无 类型：Function
说　明： layer_pc 组件一个按钮的提示框--通知框 
		layerPc.alert({
		   title: '标题',
		   content: '内容',
		   btn: ['知道了'],
		   call: function(){
		      alert('底部操作按钮的回调函数');
		   }
		});
layerPc.confirm(params) - 两个按钮的提示框--询问框
参数一：
类　型：Object
解　释：该参数有8个属性
1.不变的属性（不需要传值）: skin/area 使用自带的默认值即可
2.title: 弹窗的标题 默认：弹窗标题 类型：String
3.content: 弹窗的内容 默认：主体内容 类型：String/DOM
4.btn: 弹窗的底部操作按钮，数组传递两个值即可。会根据数组传递的顺序从左到右去生成按钮 默认：["确定", "取消"] 类型：Array
5.call: 弹窗的底部操作左边按钮的回调函数 默认：无 类型：Function
6.callSec: 弹窗的底部操作右边按钮的回调函数 默认：无 类型：Function
7.isShut: 点击左边按钮控制弹窗是否关闭（根据具体情况进行传值，如果默认点击关闭的话则无需传值） 默认：false 类型：Bollean
说　明： layer_pc 组件两个按钮的提示框--询问框，用户可以根据自己的意愿去选择性的点击。 
		layerPc.confirm({
		   title: '标题',
		   content: '内容',
		   btn: ['同意', '不同意'],
		   call: function(){
		      alert('同意');
		   },
		   callSec: function(){
		      alert('不同意');
		   },
		   isShut: true
		});
layerPc.ask(params) - 两个按钮的提示框--询问框
参数一：
类　型：Object
解　释：该参数有6个属性
1.不变的属性（不需要传值）: skin/area 使用自带的默认值即可
2.title: 弹窗的标题 默认：提示 类型：String
3.tip_ask_con: 弹窗的内容的提示语 默认：确定要删除吗？ 类型：String
4.btn: 弹窗的底部操作按钮，数组传递两个值即可。会根据数组传递的顺序从左到右去生成按钮 默认：["确定", "取消"] 类型：Array
5.call: 弹窗的底部操作左边按钮的回调函数 默认：无 类型：Function
说　明： layer_pc 组件两个按钮的提示框--询问框，用户可以根据自己的意愿去选择性的点击。该方法跟layerPc.confirm有点类似，但是这个方法是有固定格式的。格式是：带有内容为文本的字符串 
		layerPc.ask({
		   title: '标题',
		   tip_ask_con: '内容提示语',
		   btn: ['同意', '不同意'],
		   call: function(){
		      alert('同意');
		   }
		});
layerPc.success(params) - 操作回馈层/信息提示层
参数一：
类　型：Object
解　释：该参数有7个属性
1.不变的属性（不需要传值）: skin/area 使用自带的默认值即可
2.title: 弹窗的标题 默认：提示 类型：String
3.tip_ok_con: 弹窗的内容的提示语 默认：操作成功 类型：String
4.tip_type: 提示的类型 0:成功 1:警告 2:失败 默认：0 类型：Number
5.call: 层销毁后的回调函数 默认：无 类型：Function
6.time: 层自动销毁的时间 默认：2000 类型：Number
说　明： layer_pc 组件操作回馈层/信息提示层，有三种状态提供 
		layerPc.success({
		   title: '标题',
		   tip_ok_con: '失败',
		   tip_type: 2,
		   time: 3000,
		   call: function(){
		      alert('信息提示即将消失了');
		   }
		});
layerPc.frame(params) - 弹窗包含iframe层
参数一：
类　型：Object
解　释：该参数有7个属性
1.不变的属性（不需要传值）: skin使用自带的默认值即可
2.title: 弹窗的标题 默认：页面层的标题 类型：String
3.area: 弹窗的宽度和高度，第一个值是宽度，第二个值是高度。 默认：["525px", "400px"] 类型：Array
4.url: iframe的url 默认：无 类型：String
5.frame_yes_btn:底部左边按钮 左边按钮的id值 默认：#frame_btn_yes 类型：String 这个按钮务必在页面里面体现以下，即便没有用到也要存在该Dom元素。只怪当初太年轻，封装的太次了！！！
6.frame_no_btn:底部右边按钮 右边按钮的id值 默认：#frame_btn_no 类型：String 这个按钮务必在页面里面体现以下，即便没有用到也要存在该Dom元素。只怪当初太年轻，封装的太次了！！！
7.call: 点击左边按钮的回调函数 默认：无 类型：Function
说　明： layer_pc 组件弹窗包含iframe层，使得iframe可以加载指定的url页面 
		layerPc.frame({
		   title: '标题',
		   url: 'http://www.baonahao.com',
		   
		});
